# REPO_INDEX

Индекс репозитория `kassya20/ut` для быстрого анализа выгрузки конфигурации 1С.

Сформировано по текущей ветке `main` после коммита `81c5424ab34f94a0a988ae78a5635fa6171983e5` (`Добавлены файлы базы УТ`).

## Назначение индекса

Файл нужен, чтобы быстро находить объекты конфигурации при анализе без повторного обхода дерева GitHub.

Текущий фокус анализа:

- регистры накопления;
- регистры сведений, если они есть в выгрузке;
- связанные XML-описания объектов;
- модули менеджера и набора записей для регистров.

## Обнаруженные верхнеуровневые элементы

```text
.gitignore
AccumulationRegisters/
REPO_INDEX.md
```

На момент формирования индекса в просмотренном коммите обнаружены XML-объекты типа `AccumulationRegister`. XML-объекты типа `InformationRegister` в прочитанном diff коммита не обнаружены; если каталог `InformationRegisters/` будет добавлен позже, индекс нужно дополнить.

## Слой анализа для будущих проверок

### Слой 0. Карта регистров

Цель: понять, какие регистры есть, какие аналитики и ресурсы они содержат.

### Слой 1. Один регистр

Проверки по одному регистру в разрезе полной аналитики регистра:

```text
отрицательные остатки;
отрицательные обороты там, где они невозможны по смыслу;
сумма без количества;
количество без суммы;
пустые ключевые аналитики в движениях;
движения по помеченным объектам;
аномальные движения по периоду/регистратору.
```

### Слой 2. Связанные регистры

Проверки согласованности между регистрами:

```text
деньги отражены, но не закрыли расчеты;
расчеты отражены, но денежного движения нет;
движение по контрагенту есть, но нет связанного движения по деньгам/доходам/расходам;
движение по номенклатуре есть, но нет связанного движения по доходам/расходам;
НДС/акцизы есть без базового движения или наоборот.
```

### Слой 3. Backlog: зависит от настроек

```text
обязательность серий;
обязательность характеристик;
ордерная схема;
адресное хранение;
обособленное обеспечение;
правила резервирования;
настройки видов номенклатуры;
учетные политики и функциональные опции.
```

На первом этапе эти проверки не выполняются.

## Первичные группы проверок по текущему набору регистров

### Денежные средства

Регистры:

```text
ДвиженияДенежныхСредств
ДвиженияДенежныеСредстваКонтрагент
ДвиженияДенежныеСредстваДоходыРасходы
```

Потенциальные проверки:

```text
движение денежных средств с пустой ключевой аналитикой;
расхождение движения денег и движения по контрагенту;
расхождение движения денег и движения по доходам/расходам;
отрицательные/аномальные суммы по аналитике;
движения будущим периодом;
движения по помеченным объектам.
```

### Доходы, расходы, активы, пассивы

Регистры:

```text
ДвиженияДоходыРасходыПрочиеАктивыПассивы
ДвиженияКонтрагентДоходыРасходы
ДвиженияНоменклатураДоходыРасходы
ДвиженияПоПрочимАктивамПассивам
```

Потенциальные проверки:

```text
движение по доходам/расходам без связанного объекта аналитики;
движение по контрагенту без дохода/расхода;
движение по номенклатуре без дохода/расхода;
сумма без связанной бизнес-аналитики;
расхождение сумм между связанными регистрами.
```

### Номенклатура

Регистры:

```text
ДвиженияНоменклатураДоходыРасходы
ДвиженияНоменклатураНоменклатура
```

Потенциальные проверки:

```text
движение номенклатуры без номенклатурной аналитики;
расхождение парных движений номенклатура-номенклатура;
движения по помеченной номенклатуре;
сумма/количество в разных аналитиках;
аномальные движения по полной аналитике.
```

### Контрагенты

Регистры:

```text
ДвиженияДенежныеСредстваКонтрагент
ДвиженияКонтрагентДоходыРасходы
ДвиженияКонтрагентКонтрагент
```

Потенциальные проверки:

```text
движения по помеченному контрагенту;
движение без контрагента при наличии контрагентской аналитики;
расхождение парных движений контрагент-контрагент;
движение по контрагенту без связанного денежного/доходного движения.
```

### Налоги и акцизы

Регистры:

```text
ДвиженияПоНДС
ДвиженияПоАкцизам
```

Потенциальные проверки:

```text
НДС без базового движения;
акциз без базового движения;
отрицательные/аномальные суммы;
пустые ключевые аналитики;
расхождение налоговых движений с доходами/расходами.
```

### Бонусные баллы

Регистры:

```text
БонусныеБаллы
```

Потенциальные проверки:

```text
отрицательный остаток бонусных баллов по полной аналитике;
остаток бонусов по помеченному объекту аналитики;
движения бонусов без ключевой аналитики;
старые неиспользуемые остатки бонусов.
```

## Следующий шаг анализа

1. Последовательно открыть XML каждого регистра.
2. Выписать измерения, ресурсы, реквизиты и тип регистра.
3. На основе фактических аналитик составить реестр проверок.
4. Не использовать настройки программы и учетные политики на первом этапе.
